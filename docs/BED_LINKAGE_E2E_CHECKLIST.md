# 床位联动 E2E 检查清单

## 目标

验证“床位管理管理”里的数据变更，是否能自动同步到：

- 长者管理 -> 床态全景
- 床位管理 -> 房态图（床位全景）

## 前置条件

- 已启动前端和后端服务。
- 当前账号有床位管理、长者管理访问权限。
- 至少准备 1 个楼栋、1 个楼层、1 个房间、1 个床位。

## 场景 1：管理页修改房间后，长者床态全景自动刷新

1. 打开“床位管理”，进入“房间管理”。
2. 修改一个房间的楼层或房型并保存。
3. 立刻切到“长者管理 -> 床态全景”。
4. 预期：
   - 无需手动刷新页面即可看到更新后的房间信息。
   - 矩阵中楼栋/楼层/房间映射关系正确。

## 场景 2：管理页修改床位后，长者床态全景自动刷新

1. 打开“床位管理”，进入“床位管理”。
2. 修改床位状态（空床/入住/维修）并保存。
3. 切到“长者管理 -> 床态全景”。
4. 预期：
   - 床位颜色、状态标签与管理页一致。
   - 统计卡片（空闲/在住/维修）自动变化。

## 场景 3：管理页修改房间/床位后，床位全景自动刷新

1. 打开“床位管理 -> 房态图”。
2. 在另一页签打开“床位管理 -> 床位管理”并修改房间或床位。
3. 回到房态图页签观察。
4. 预期：
   - 房态图在短时间内自动更新，无需手工刷新。
   - 列表视图和二维矩阵视图数据一致。

## 场景 4：删除保护与错误提示

1. 尝试删除“存在下级数据”的楼栋、楼层、房间。
2. 尝试删除“已绑定长者”的床位。
3. 预期：
   - 后端拒绝删除。
   - 前端弹出明确错误原因，不是笼统“删除失败”。

## 场景 5：负向验证（无关模块不触发刷新）

1. 保持“床态全景”或“房态图”打开。
2. 在 OA/营销等无关模块执行保存操作。
3. 预期：
   - 床态相关页面不发生无意义刷新。
